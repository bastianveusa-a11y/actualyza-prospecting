Meta Ads Library API Integration

Actualyza is a B2B intelligence platform for the healthcare and wellness industry. We use the Meta Ads Library API (ads_archive endpoint with ads_read permission) to identify which dental, aesthetic, medical spa, and wellness clinics in the United States are actively running ads on Meta platforms — helping us prioritize outreach for our AI receptionist product.

How the ads_read permission is used
STEP 1

Clinic Discovery

We identify clinics via Google Places API by category (dental, esthetic, medspa, wellness) and city (Miami, Orlando, Dallas).

STEP 2

Meta Ads Lookup

For each clinic, we query the Meta Ads Library API to check if they have active ads in the US. We use the ads_read permission to access this public data.

GET /ads_archive
STEP 3

Prioritized Outreach

Clinics with active Meta ads are scored higher — they're already investing in digital marketing and are more likely to be interested in our AI solution.

Sample API call — ads_archive endpoint
GET graph.facebook.com/v21.0/ads_archive
// Request parameters sent to Meta Ads Library API { "access_token": "[USER_ACCESS_TOKEN with ads_read]", "ad_reached_countries": ["US"], "ad_active_status": "ACTIVE", "search_terms": "Miami Dental Studio", "limit": 50, "fields": "id" } // Response — we only store the count, not the ad content { "data": [ { "id": "..." }, { "id": "..." } ], "paging": { "cursors": { ... } } } // We extract: count of active ads → classify as High / Medium / Low / None

Data usage policy: Actualyza only retrieves the count of active ads — we do not store ad creative content, targeting parameters, audience data, or any personal information. The ad count is used solely to calculate a marketing activity score that helps us prioritize which businesses to contact for our B2B outreach. Data is not shared with third parties.
